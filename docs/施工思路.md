# UI预览工具 - 施工思路

## 项目目标

创建一个基于2560×1440基准分辨率的UI预览工具，所有UI元素以该分辨率为设计基准，实现自适应缩放、硬阴影效果和可编辑配置。

## 核心设计原则

### 1. 基准分辨率自适应
- **基准分辨率**: 2560×1440
- **自适应逻辑**: 所有UI元素的位置和尺寸基于基准分辨率进行设计，在不同分辨率下按等比缩放
- **缩放公式**: `scale = Math.min(window.innerWidth / baseWidth, window.innerHeight / baseHeight)`
- **应用方式**: 将基准像素值乘以缩放比例得到实际像素值

### 2. 硬阴影系统
- **硬阴影定义**: 位于UI元素下方，向右下角偏移的纯色阴影（非模糊阴影）
- **阴影层级**: 阴影层(`z-index`低于本体)，本体层(`z-index`高于阴影)
- **可配置参数**:
  - X偏移量: 向右偏移的像素数
  - Y偏移量: 向下偏移的像素数  
  - 颜色: 阴影颜色（支持透明色）
  - 透明度: 阴影透明度

### 3. 模块化架构
- **分离关注点**: HTML、CSS、JavaScript分离
- **功能模块化**: 按功能将JavaScript拆分为独立模块
- **配置集中**: 使用JSON配置文件和`localStorage`进行状态持久化

### 关键技术点

#### 1. 自适应缩放实现
```javascript
class Scaler {
    static BASE_WIDTH = 2560;
    static BASE_HEIGHT = 1440;
    
    static getScale() {
        return Math.min(
            window.innerWidth / this.BASE_WIDTH,
            window.innerHeight / this.BASE_HEIGHT
        );
    }
    
    static toActual(baseValue) {
        return baseValue * this.getScale();
    }
}
```

#### 2. 阴影效果实现
```css
.ui-element-shadow {
    position: absolute;
    z-index: 1; /* 低于主层的z-index */
    background-color: rgba(0, 0, 0, 0.5);
    opacity: 0.7;
}
```

#### 3. 配置持久化
```javascript
class Storage {
    static saveConfig(key, config) {
        localStorage.setItem(key, JSON.stringify(config));
    }
    
    static loadConfig(key, defaultConfig) {
        const saved = localStorage.getItem(key);
        return saved ? JSON.parse(saved) : defaultConfig;
    }
}
```

## 扩展性设计

### 新增UI元素流程
1. 创建新的组件类（如`ButtonGroupUI.js`）
2. 扩展`ui-config.json`中的元素定义
3. 在`UIManager`中注册新组件
4. 在编辑器中添加相应的编辑控件

## 注意事项

1. **性能优化**: 缩放计算使用防抖，避免频繁重排
2. **兼容性**: 确保在主流浏览器中正常工作
3. **用户体验**: 编辑操作要直观，提供即时反馈
4. **代码质量**: 遵循模块化原则，确保代码可维护性
5. **文档完整**: 所有模块都需要清晰的注释和使用说明

## 验收标准

1. 状态栏在2560×1440分辨率下显示为2000×200像素
2. 在1920×1080分辨率下等比缩放
3. 具有可配置的硬阴影效果
4. 编辑模式可直接修改状态栏位置、尺寸和阴影
5. 配置可保存并在刷新后恢复
6. 代码结构清晰，模块分离良好
