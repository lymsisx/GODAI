# UI预览工具 - 系统架构说明

## 项目概述

UI预览工具是一个基于Web的像素级UI布局预览系统，专门用于游戏UI的设计和验证。系统采用模块化架构，支持基于2560×1440基准分辨率的自适应缩放，并提供可视化的编辑功能。

## 核心特性

1. **像素级缩放系统** - 基于2560×1440设计分辨率，自动计算缩放比例
2. **多层UI元素系统** - 支持本体+阴影的多层结构
3. **实时编辑器** - 提供位置、尺寸、阴影等参数的实时编辑
4. **配置持久化** - 使用localStorage保存用户配置
5. **模块化架构** - 代码按功能模块组织，易于维护和扩展

## 核心模块说明

### 1. Scaler.js - 缩放计算工具
- **功能**：处理基于2560×1440基准分辨率的自适应缩放
- **主要方法**：
  - `getScale()` - 获取当前缩放比例
  - `toActual(baseValue)` - 基准像素转实际像素
  - `toBase(actualValue)` - 实际像素转基准像素
  - `scalePosition(baseX, baseY)` - 缩放位置坐标
  - `scaleSize(baseWidth, baseHeight)` - 缩放尺寸

### 2. Storage.js - 本地存储工具
- **功能**：管理localStorage的读写操作
- **主要方法**：
  - `save(key, data)` - 保存数据
  - `load(key, defaultValue)` - 加载数据
  - `remove(key)` - 删除数据
  - `clear()` - 清空所有数据

### 3. UIElementBase.js - UI元素基类
- **功能**：所有UI元素的基类，定义通用接口
- **属性**：id, x, y, width, height, shadow配置
- **方法**：
  - `render(scale)` - 渲染UI元素
  - `update(config)` - 更新配置
  - `getConfig()` - 获取当前配置
  - `destroy()` - 销毁元素

### 4. StatusBarUI.js - 状态栏组件
- **功能**：状态栏UI元素的实现
- **特性**：
  - 多层结构（本体+阴影）
  - 硬阴影系统（可调整偏移和颜色）
  - 基于基准分辨率的自适应布局
- **默认配置**：x:280, y:20, width:2000, height:200

### 5. UIManager.js - UI管理器
- **功能**：管理所有UI元素的创建、更新和销毁
- **特性**：
  - 元素注册和注销
  - 批量渲染和更新
  - 窗口大小改变时的自动重新渲染
  - 配置持久化管理

### 6. StatusBarEditor.js - 状态栏编辑器
- **功能**：提供状态栏的可视化编辑功能
- **编辑参数**：
  - 位置（X, Y坐标）
  - 尺寸（宽度, 高度）
  - 阴影（偏移X, 偏移Y, 颜色）
- **操作**：应用、保存、重置、实时预览

### 7. App.js - 主应用类
- **功能**：整合所有模块，管理应用生命周期
- **流程**：
  1. 初始化缩放系统
  2. 加载配置
  3. 初始化UI管理器
  4. 创建UI元素
  5. 初始化编辑器
  6. 设置事件监听
  7. 首次渲染

### 8. main.js - 应用主入口
- **功能**：模块加载和应用启动
- **特性**：
  - 模块加载器（确保加载顺序）
  - 应用启动器
  - 错误处理和恢复

## 数据流

```
用户操作 → 编辑器 → 更新UI元素配置 → UIManager重新渲染
                ↓
           Storage保存配置 → 页面刷新时自动加载
```

## 扩展指南

### 添加新的UI元素类型
1. 在 `js/ui/` 目录下创建新的UI组件类（继承 `UIElementBase`）
2. 在 `UIManager` 中注册新元素类型
3. 在 `config/default-config.json` 中添加默认配置
4. 在编辑器中添加对应的编辑控件

### 添加新的编辑功能
1. 在 `js/editor/` 目录下创建新的编辑器类
2. 在 `App.js` 中初始化新编辑器
3. 在 `index.html` 的编辑面板中添加控件

### 修改基准分辨率
1. 修改 `Scaler.js` 中的 `BASE_WIDTH` 和 `BASE_HEIGHT`
2. 更新 `config/default-config.json` 中的基准分辨率
3. 重新计算所有UI元素的基准位置和尺寸

## 开发规范

### 代码规范
- 使用ES6+语法和类
- 模块化设计，单一职责原则
- 详细的JSDoc注释
- 错误处理和恢复机制

### 命名规范
- 类名：大驼峰式（如：`StatusBarUI`）
- 方法名：小驼峰式（如：`getScale()`）
- 文件名：与类名一致（如：`StatusBarUI.js`）
- CSS类名：kebab-case（如：`ui-element-container`）

### 提交规范
- 每次修改后更新相关文档
- 测试所有相关功能
- 检查代码语法和格式

## 故障排除

### 常见问题
1. **模块加载失败**：检查文件路径和加载顺序
2. **缩放计算错误**：检查窗口大小和基准分辨率
3. **配置保存失败**：检查localStorage是否可用
4. **UI渲染异常**：检查CSS样式和DOM结构

### 调试工具
- 控制台输入 `App.getStatus()` 查看应用状态
- 控制台输入 `Scaler.getScaleInfo()` 查看缩放信息
- 使用 `test-modules.html` 进行模块测试
